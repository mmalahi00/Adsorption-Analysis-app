# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 2.0.x   | :white_check_mark: |
| < 2.0   | :x:                |

## Reporting a Vulnerability

Please report security vulnerabilities through one of these methods:

1. **GitHub Security Advisory**: Use the "Report a vulnerability" button in the Security tab (preferred)
2. **Private Issue**: Open a private security issue if available
3. **Email**: Contact the maintainers via the email listed in their GitHub profiles

**Do not** open public issues for security vulnerabilities.

### What to Include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

### Response Timeline

- **Initial Response**: Within 48 hours
- **Status Update**: Within 7 days
- **Resolution Target**: Within 30 days for critical issues

We appreciate responsible disclosure and will acknowledge contributors in our release notes (unless you prefer to remain anonymous).
